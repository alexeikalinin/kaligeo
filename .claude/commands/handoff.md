# /handoff — Session Handoff

Запускай в конце каждой рабочей сессии. Обновляет SESSION.md актуальным состоянием проекта.

## Что делать

1. Прочитай текущий SESSION.md (если есть)
2. Проанализируй что было сделано в этой сессии (git log, изменения файлов, контекст разговора)
3. Перезапиши SESSION.md по шаблону ниже
4. Закоммить SESSION.md (он коммитится автоматически через хук)

## Шаблон SESSION.md

```markdown
# SESSION — KaliGEO Landing

_Обновлено: {{дата и время}}_

## Статус проекта
{{одна строка: что сейчас представляет собой проект}}

## Последнее сделано
{{bullet-список: что было сделано в последней сессии, конкретно}}

## В процессе / незавершено
{{что начато но не закончено, что нужно проверить}}

## Отложено / backlog
{{идеи и задачи которые решили отложить, с кратким пояснением почему}}

## Следующие шаги
{{конкретные задачи которые логично делать следующими, по приоритету}}

## Технические детали для контекста
{{ключевые решения, API-ключи (не сами ключи!), внешние сервисы, нюансы реализации}}

## Открытые вопросы
{{что не решено, что нужно уточнить у пользователя}}
```

## Правила

- Пиши конкретно: не "обновили форму" а "подключили Web3Forms, ключ в hidden input, письма идут на alexeikalinin1@yandex.ru"
- Раздел "Следующие шаги" — максимум 5 пунктов, только реалистичные
- Если раздел пустой — пиши `—` не оставляй пустым
- После записи скажи пользователю: "SESSION.md обновлён, до следующего раза 👋"
